# ithub-frontend-vue-nuxt

_(контрольная точка №4)_

## Задание

Перенести приложение, разработанное в рамках *контрольной точки №2* по [макету](https://www.figma.com/file/NfRkFdCH22pMltWMmjdDRP/Auth-pages-tamplate?type=design&node-id=0%3A1&mode=dev&t=xV0MRNPCqesL0tR5-1), на фреймворк `Nuxt 4` с использованием библиотек 
`better-auth` и `kysely`.

В текущем репозитории уже реализован полный либо частичный перенос страниц __Home__ и __Users__, а также присутствует серверная часть в минимальном объеме. 

На основе данного репозитория необходимо:

1. Добавить страницу __Login__ и соответствующую серверную логику. Логин производить в `/server/api/login.post.ts`, используя функцию `auth.api.signInEmail` (**2 балла**). 

2. В случае успешного логина перенаправлять пользователя на страницу Профиля (**1 балл**)

2. Настроить соединение и перенести данные из `users.json` в базу данных `sqlite3`. Работу вести в `server/db/seed.ts`. Для занесения основной информации о пользователях использовать `auth.api.signUpEmail`, для добавления дополнительных полей с биографией использовать `kysely` либо `better-auth`. (**2 балла**)

3. Доработать страницу __Users__ и добавить соответствующую серверную функцию, чтобы получать данные из базы (через `kysely` или через админский плагин для `better-auth`. (**2 балла**)

4. Перенести и доработать страницу __Profile__ и соответствующую серверную логику. Страница должна получать информацию о текущем залогиненном пользователе, подтягивая дополнительные поля. (**2 балла**)

5. Ссылки в навигации должны различаться в зависимости от того, залогинен ли пользователь (**1 балл**)

6. Реализовать процедуру выхода (логаут) как на фронте, так и на сервере. (**1 балл**)
